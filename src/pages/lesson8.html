Get Started with JSON Web Tokens

JSON web token (JWT) defines a way of sending and receiving information as JSON object between parties. The object is signed digitally and so can be verified and trusted. There are two ways to sign a JWT; a secret using HMAC algorithm or a public or private key pair using RSA. JWTs are compact because they are small in size and can be sent over a URL, POST parameter or inside the HTTP header. They are also self-contained because all the information required about the user are contained in the payload.

We use JWTs for authentication and information exchange. JWTs contains the header, the payload and the signature. The header has the token type and the hashing algorithm type. The payload contains the claims or the information that is being sent about the entity. The signature is created by taking the encoded header, the encoded payload, a secret and the algorithm specified in the header, and sign.

In authentication, when users log in their credentials, a JWT is returned. This should not be kept longer than required to avoid running into security compromise. JWT is less worded than XML and smaller when encoded. In terms of security, while Simple Web Tokens (SWT) can only be signed by shared secret using HMAC algorithm, JWT can also use a private/public key pair.
