HTML5 APIs

HTML5 allows different feature to be developed independently at their own pace and be implemented without having to wait for other features to be completed. Modules are in different stage from conception to development and to full implementation. caniuse.com allows us to see if a specific feature is ready for use.

The data- attribute allows data to be embedded in a page using custom attributes ignored by the browser. The sole purpose of the embedded data is to be used by a specific JS program. While the name of the attributes can be decided by the developer, they must start with “data-“, contain only lowercase letters, numbers, hyphens, dots, colons or underscores. They may also contain a string value. It could be something like; data-name = “spider man” data-superpower = “flying” data-rating = “6” data-group = “marvel” The data- attribute is supported by most modern browsers. We can work around the string-only value by first encoding the value as a JSON string and performing a type conversion later.

HTML5 has a number of APIs that helps gain access to hardware such as cameras, batteries, geolocation and so on. The HTML5 web storage API provides a key-value store on the client, similar to cookies but with fewer restrictions. The web storage API has a storage capacity much more than 4kb for cookies, makes information available in multiple windows, keeps data validity even after cookies have expired. The window object of a browser that supports the web storage API will have the localStorage property. sessionStorage works the same way but only for the current session (works like cookies?)

The geolocation API is used to get the geographical location of the device, meaning we can use to find the client’s exact location. We can also use the information to link nearby locations and be able to measure the client’s speed and direction of travel. If supported by the browser, there will be a geolocation property of the navigator object. This geolocation property has the getCurrentPosition() method that will return a position object to a specified callback function called youAreHere(). Passing the position object to the youAreHere() function returns a longitude and latitude property.

The Worker() constructor function creates a new worker that allows JS program to run smoothly avoiding the dreaded “unresponsive script” message. Websocket is a new protocol that allows two-way communication between a client and the server. Once established, the connection is kept open and responses are pushed to the client once received from the server. The notification API allows us to show messages using the device’s inbuilt notification system. The require the user’s permission to work and permission can be requested by using the requestPermission() method of the notification global object.
